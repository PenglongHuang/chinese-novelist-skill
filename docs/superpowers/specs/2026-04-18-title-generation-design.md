# Phase 1 Layer 3 — 小说标题生成 设计文档

## 概述

在 Phase 1（分层问答）中新增 Layer 3：标题生成阶段。在 Layer 1（核心定位）和 Layer 2（深度定制）收集完创意信息后，AI 基于收集到的故事元素自动生成 3-5 个候选标题，用户从中选择或自行提供，选定标题自动贯穿后续所有流程。

## 背景与动机

当前工作流缺少专门的标题创作环节。标题是小说的第一印象，影响读者点击意愿和作品传播。一个独立且专业的标题生成阶段可以：

- 基于已收集的故事元素（题材、冲突、角色、主题）进行针对性创作
- 按题材风格自动适配标题风格，避免千篇一律
- 让标题选择成为一个有意识的创意决策，而非事后补充

## 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 阶段位置 | Phase 1 Layer 3 | 符合现有分层问答架构，不改后续编号 |
| 生成方式 | AI 生成 + 用户选择 | 平衡创意性与用户控制力 |
| 风格策略 | 按题材自动适配 + 独立指南文件 | 可维护、可扩展，符合 flows/guides 分层模式 |
| 标题传递 | 通过 `AskUserQuestion` 结果获取，存入对话上下文，由 Phase 2 写入文件 | 复用现有交互模式，无需额外持久化机制 |
| 迭代次数 | 无限制（超过 5 轮后软提示用户自定义） | 给用户充分的选择空间，同时避免无限循环 |

## 阶段定位

```
Phase 0（初始化）
    ↓
Phase 1 Layer 1（核心定位 — 必选，3 题）
    ↓
Phase 1 Layer 2（深度定制 — 可选，5 题）
    ↓
Phase 1 Layer 3（标题生成）  ← 新增
    ↓
Phase 2（规划与确认，含写作模式选择）
    ↓
Phase 3（自动化写作）
    ↓
Phase 4（校验与修复）
```

## 执行流程

### Step 1: 分析创意元素

从 Phase 1 Layer 1 和 Layer 2 收集的信息中提取关键元素：

- 题材类型（悬疑、言情、奇幻、科幻、历史、都市、惊悚等）
- 核心冲突（故事的主要矛盾）
- 主角特征（身份、性格、目标）
- 核心主题（深层表达）
- 氛围/基调（压抑、温暖、紧张、浪漫等）

根据题材类型，匹配 `title-guide.md` 中对应的风格规则。

### Step 2: 生成候选标题

根据匹配的风格规则，运用 `title-guide.md` 中的创作技巧，生成候选标题：

- 简单/单一题材故事：生成 3 个候选
- 复杂/跨题材故事：生成 5 个候选
- 每个候选使用不同的创作技巧，避免风格雷同

每个标题附带简短说明：

- 使用了什么创作技巧
- 为什么适合这个故事

### Step 3: 展示与选择（使用 AskUserQuestion）

使用 `AskUserQuestion` 工具展示候选标题，每个标题作为一个 Option。额外提供一个"重新生成"选项。用户也可以通过 "Other" 自定义标题。

交互结构：

```
AskUserQuestion:
  question: "基于您的故事元素，以下是为您生成的候选标题，请选择："
  options:
    - label: "《雾中灯塔》"
      description: "意象隐喻法 — 以'灯塔'暗示主角在迷茫中的指引者角色"
    - label: "《第七个失踪者》"
      description: "悬念留白法 — 用数字制造悬念，暗示案件核心"
    - label: "《她最后一次微笑》"
      description: "反差法 — '微笑'与失踪形成情感张力"
    - label: "《迷雾档案》"
      description: "核心冲突提炼法 — 直指故事核心的悬疑调查线"
    - label: "重新生成"
      description: "不满意这些标题，重新生成一组新的候选"
  multiSelect: false
```

用户通过 "Other" 可自行输入标题。选择"重新生成"后回到 Step 2 重新生成。

**迭代策略**：超过 5 轮重新生成后，在展示时追加软提示："您已尝试多轮，也可以直接输入您心中的标题。"，引导用户自定义。

### Step 4: 确认标题并过渡

用户确认最终标题后：

1. **展示确认信息**：
   ```
   ✅ 小说标题已确定：《[标题]》
   接下来将进入规划阶段，为您生成大纲和人物档案。
   ```

2. **标题信息传递**：标题存入当前对话上下文。Phase 2 规划阶段启动时，从上下文中读取标题，用于：
   - 命名项目目录（`{timestamp}-{标题}/`）
   - 写入 `03-写作计划.json` 的 `novelName` 字段
   - 写入 `00-大纲.md` 的文件头

3. **中断恢复**：标题仅在对话上下文中传递，不单独持久化。若用户在 Layer 3 后、Phase 2 前中断会话：
   - Phase 0 检测到未完成项目时，重新进入 Layer 3 让用户重新选择标题
   - 这是可接受的，因为标题选择耗时很短

## 标题生成指南（title-guide.md）

### 题材-风格映射

指南按题材类型定义标题风格特征：

| 题材 | 风格特征 | 常用结构 |
|------|----------|----------|
| 悬疑/推理 | 悬念感、未解之谜 | "谜、局、消失的X、第N个X" |
| 言情/甜宠 | 诗意/温暖感 | 意象、四字短语、对比结构 |
| 奇幻/仙侠 | 宏大/古风感 | "录、传、纪、诀、歌" |
| 科幻 | 未来感/哲思感 | 技术意象、悖论结构 |
| 历史 | 厚重感/时代感 | 年代、地名、事件锚点 |
| 都市/现实 | 接地气/生活化 | 口语化表达、反差结构 |
| 惊悚/恐怖 | 压迫感/禁忌感 | 暗示、未完成句式 |

### 创作技巧

1. **核心冲突提炼法**：从故事核心冲突中提炼关键词组合成标题
2. **主角命名法**：以主角特征或名字为核心构建标题
3. **意象隐喻法**：用核心意象暗示主题，留有解读空间
4. **反差法**：在标题层面制造认知反差，引发好奇心
5. **悬念留白法**：标题不揭示全部，留下想象空间

### 质量标准

- 字数建议：2-8 字为主流，特殊题材可到 10 字
- 避免 AI 套路化表达（"XX之道""XX的觉醒""XX纪元"等）
- 避免与知名作品标题雷同
- 朗朗上口、易于记忆和传播
- 每个候选标题应使用不同的创作技巧，避免风格雷同
